# ADR 0001: Вибір архітектурного стилю

## Контекст
Ми розробляємо MVP системи бронювання столиків. Команда невелика, строки стислі. Необхідно обрати архітектурний стиль для старту.

## Рішення
Ми обираємо **Модульний Моноліт**.
Весь код (API, бізнес-логіка, домен) буде знаходитися в одному репозиторії та запускатися як один процес Node.js.

## Альтернативи
* Мікросервіси (занадто складно для старту, overhead на інфраструктуру).

## Наслідки
* + Простий деплой та налагодження.
* + Швидка розробка.
* - При зростанні навантаження доведеться масштабувати весь додаток цілком.
